# aimani-gs

G's Academy版MVPアイマニ — 受講生がAIと困りごとを整理し、チューター・メンターに安全に相談できるサービス。

## コンセプト

G's Academyの受講生が、課題・チーム開発・進路のモヤモヤを、いきなりチューターやメンターに投げる前に、AIと一緒に言葉にして、相談しやすい形に整えるサービス。

### アイマニAIとの関係

このプロダクトは、アイマニAI（職場の困りごとを安全に言葉にするサービス）のG's Academy版プロトタイプ。G'sという閉じた環境で、アイマニの設計思想に需要があるかを検証する。

### 解決する課題

- 「課題で詰まっているけど、何を聞けばいいか分からない」
- 「Slackで聞くのが怖い」
- 「こんな初歩的な質問をしていいのか迷う」
- 「チーム開発で困っているけど、誰にどう言えばいいか分からない」
- 「メンター面談で話したいことがあるけど、うまく言葉にできない」

### AIの役割

- 判断しない。答えを出さない
- 困りごとを具体的な問いで引き出す（掘る・広げる）
- 材料を並べて、本人が自分の言葉を掴むのを手伝う
- 共有するかどうか、何を共有するかは本人が決める

## 技術スタック

bs-job-board（MAA39/gs-20260620-bs-job-board）の3 Workers分離構成を継承。

| 要素 | 選定 |
|---|---|
| フロント | TanStack Start on CF Workers |
| API | Hono on CF Workers |
| DB | Cloudflare D1 |
| AI | Cloudflare Workers AI（gpt-oss-120b + qwen3-30b） |
| 認証 | Better Auth + GitHub OAuth |
| モノレポ | Turborepo + pnpm |

## ロール

| ロール | 説明 |
|---|---|
| student | G's受講生。相談を作成し、AIと整理する |
| tutor | チューター。相談を閲覧し、コメントする |
| mentor | メンター。相談を閲覧し、コメントする |
| ai | AI。材料を並べ、言語化を支援する |

## 設計原則（アイマニAIから継承）

1. **内発的動機で話す → 副産物で情報が流れる。** 受講生が自分のために整理する。副産物でチューター/メンターに課題が見える
2. **報告コストゼロ。** 相談するだけ。レポートを書かなくていい
3. **信頼を前提にしない → 結果にする。** 「相談していいですか」と聞かなくていい。AIがまず受け止める
4. **プロダクト消えても不利にさせない。** 相談内容が評価に使われない
5. **選択肢 + 自由記述。** AIが選択肢を出す + 自由に書ける
