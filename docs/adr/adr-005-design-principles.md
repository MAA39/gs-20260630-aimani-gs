# ADR-005: アイマニ設計原則のG's版適用

**状態:** 確定
**日付:** 2026-06-30

## 決定

アイマニAIの5つの設計原則をG's版に適用する。

## 適用

| # | 原則 | G's版での意味 |
|---|---|---|
| 1 | 内発的動機で話す→副産物で情報が流れる | 受講生が自分のために整理→副産物でチューター/メンターに課題が見える |
| 2 | 報告コストゼロ | 相談するだけ。日報を書かなくていい |
| 3 | 信頼を前提にしない→結果にする | 「聞いていいですか」と聞かなくていい。AIがまず受け止める |
| 4 | プロダクト消えても不利にさせない | 相談内容が成績・評価に使われない |
| 5 | 選択肢+自由記述 | AIが選択肢を出す + 自由に書ける |

## 表現ルール

| 避ける分類 | 使う |
|---|---|
| 内面の開示を目的化する語 | 思っていること / 困りごと |
| 医療的な印象を与える語 | CBTアプローチでモヤモヤの言語化 |
| 開示をAIが引き出す前提の表現 | 言葉にならないものを本人が掴める形にする |
