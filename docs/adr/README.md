# ADR（Architecture Decision Records）

## 方針

- 判断したらすぐ書く。後から書くと記憶が腐る
- 「なぜそうしたか」と「なぜそうしなかったか」の両方を書く
- 変更は上書きではなく、Superseded byで新ADRを作る
